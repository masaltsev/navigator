## 5. Выявленные проблемы

### 5.1 MINOR — Soft-deleted дубликаты остались в БД

При удалении дубликатов использовался soft delete (`$model->delete()`). В таблицах остались 28 записей с `deleted_at IS NOT NULL`. Они не влияют на работу (не попадают в выборки Eloquent), но засоряют БД.

**Рекомендация:** Выполнить `forceDelete()` для soft-deleted дубликатов или оставить как есть (не критично). Команда: `ThematicCategory::onlyTrashed()->forceDelete()` и т.д.

### 5.2 MINOR — `SeederItem` в harvester не использует description/keywords

В `ai-pipeline/harvester/config/seeders.py` модель `SeederItem` содержит только `id`, `code`, `name`, `is_active`, `parent_code`. Поля `description` и `keywords` из JSON игнорируются.

**Рекомендация:** Расширить `SeederItem` для возможности использования `description`/`keywords` в промпте:

```python
class SeederItem(BaseModel):
    id: int | None = None
    code: str
    name: str
    is_active: bool = True
    parent_code: str | None = None
    description: str | None = None
    keywords: list[str] | None = None
```

### 5.3 MINOR — `base_system_prompt.py` не инжектирует description/keywords

Системный промпт выводит только `code → name`. Описания и ключевые слова пока не используются для маршрутизации ИИ.

**Рекомендация:** При формировании промпта добавить description и keywords рядом с именем термина. Пример формата:

```
- code:30 → Адаптация к роли родственника (помощь опекунам, школа ухода, выгорание)
```

Это повысит точность классификации, но увеличит размер промпта. Можно внедрить на Phase 2.

### 5.4 INFO — Порядок id записей в JSON не sequential

После удаления дубликатов id в JSON идут не последовательно (например, thematic_categories: id 1–27 не подряд, пропуски id 22–27 vs 28–39). Это не проблема — id не используется в harvester, только code.
