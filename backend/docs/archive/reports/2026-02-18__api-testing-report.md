## 5. Проблемы и рекомендации

### Критические проблемы

**Нет критических проблем.** Все основные эндпоинты работают корректно.

### Предупреждения

1. **PostGIS координаты в API Resources**
   - **Проблема**: Метод `extractCoordinates()` в `OrganizationResource` возвращает `null` для координат из PostGIS.
   - **Рекомендация**: Реализовать извлечение координат через raw SQL: `ST_X(coordinates) as lng, ST_Y(coordinates) as lat` или добавить accessor в модель `Venue`.

2. **Отсутствие данных для некоторых тестов**
   - **Проблема**: Некоторые тесты пропущены из-за отсутствия тестовых данных (например, организации с `fias_id`, события с RRule).
   - **Рекомендация**: Добавить seeders или factories для создания тестовых данных с полным набором полей.

3. **Внутренний API без аутентификации**
   - **Проблема**: Эндпоинты `/api/internal/*` не защищены аутентификацией (TODO в коде).
   - **Рекомендация**: Добавить middleware аутентификации (Sanctum или API key) перед продакшн-деплоем.

### Улучшения

1. **Расширение покрытия тестами**
   - Добавить тесты для обновления существующих организаций через `/api/internal/import/organizer`
   - Добавить тесты для событий с RRule
   - Добавить тесты для валидации событий

2. **Производительность**
   - Рассмотреть кэширование справочников (organization_types, thematic_categories)
   - Оптимизировать запросы с множественными фильтрами (возможно, добавить составные индексы)

3. **Документация API**
   - Добавить OpenAPI/Swagger спецификацию
   - Документировать все параметры фильтрации
