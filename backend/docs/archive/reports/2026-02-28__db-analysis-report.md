## Как запустить команду сбора статистики

Команда сбора статистики по БД Navigator Core: **`php artisan db:stats-report`**.

**Где выполнять:** из корня проекта бэкенда (`backend/`), при этом БД должна быть доступна (например, через `sail up` или локальный PostgreSQL).

**Пример:**
```bash
cd backend
php artisan db:stats-report
```

Вывод — JSON в stdout. Его можно сохранить в файл и использовать в отчётах:
```bash
php artisan db:stats-report > docs/reports/db-stats-$(date +%Y-%m-%d).json
```

## Рекомендации

1. **Организации:** В приоритете — **116 одобренных организаций без thematic_category и без источников**: для назначения категорий нужны данные с сайтов/соцсетей, т.е. сначала добавить и обойти источники (или обогатить из реестров), затем классифицировать. Параллельно продолжать обогащение остальных одобренных без источников/без категорий.
2. **События:** Наращивать импорт/создание событий (в т.ч. из Harvester), чтобы заполнять модель Event/EventInstance.
3. **Источники:** Разобрать 186 групп дублей по `base_url`, при необходимости связать дубли с одним организатором и деактивировать лишние источники.
4. **Площадки:** Провести слияние дублей по `fias_id` (и при необходимости по нормализованному адресу), чтобы одна площадка = один fias_id.
